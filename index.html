Mobile Scroll Animation

On mobile, you can hide the top row (logo & buttons) when scrolling down and reveal it when scrolling up. This creates a more compact navigation experience. The menu items stay visible.

<script>
// Basic usage
const navbar = document.querySelector('.navbar');
initNavbarScroll(navbar);

// With custom options
initNavbarScroll(navbar, {
    threshold: 20,        // Scroll distance before triggering (default: 10)
    mobileBreakpoint: 768 // Screen width for mobile behavior (default: 640)
});
</script>

Options: threshold (minimum scroll distance) | mobileBreakpoint (max screen width for effect)

required: Add required attribute to the input and the label will automatically show a required indicator

Consider these attributes for better UX:
autocomplete — Hints to browser what type of data field expects (name, email, cc-number, etc). Enables autofill.
enterkeyhint — Changes mobile keyboard's enter key label (search, go, next, done, send). Improves UX clarity.
inputmode — Controls which mobile keyboard appears (numeric, decimal, tel, email, url). More specific than input type.

Modal / Dialog

Native <dialog> element with accessible keyboard handling and focus trapping built-in.

Consider alternatives first
Modals interrupt user flow and should be used sparingly. Prefer:
  • Inline feedback — Show validation, confirmations, and errors where users are already looking
  • Native dialogs — Use window.confirm() or window.alert() for simple confirmations
  • Expandable sections — Use <details> for progressive disclosure without context loss

Standard Modal

Modal Title

This is a modal dialog using the native <dialog> element. It provides built-in accessibility features including focus trapping and keyboard handling.

<dialog id="my-modal" class="modal {size}">
    <div class="modal-header">
        <h3 class="modal-title">Title</h3>
        <button class="modal-close" onclick="closeModal('my-modal')">✕</button>
    </div>
    <div class="modal-body">...</div>
    <div class="modal-footer">
        <button onclick="closeModal('my-modal')">Cancel</button>
        <button>Confirm</button>
    </div>
</dialog>

<!-- Open with JavaScript -->
<button onclick="openModal('my-modal')">Open</button>

size: sm | lg | (omit for default)
variant: compact for centered alert-style modals

No-JS behavior: If rendered with the open attribute (e.g., <dialog open class="modal">), the modal displays immediately and can be closed without JavaScript via the close button or Esc key. However, showing the modal dynamically after page load requires JavaScript — call element.showModal() to open and element.close() to close. The showModal() method also enables the backdrop and focus trapping.

Helper functions: Include framework.js for convenience functions: openModal('id') opens a modal and sets up click-outside-to-close behavior automatically. closeModal('id') closes it. Both accept an ID string or element reference.

Native "Unsaved Changes" Dialog

Browsers have a built-in confirmation dialog when users try to navigate away from a page with unsaved changes. Use this instead of a custom modal — it's consistent across all sites and cannot be dismissed accidentally.

// Enable the warning when there are unsaved changes
window.addEventListener('beforeunload', (e) => {
    if (hasUnsavedChanges) {
        e.preventDefault();
        e.returnValue = ''; // Required for Chrome/Edge
    }
});

// Or use the helper function from framework.js:
const unsavedChanges = trackUnsavedChanges();

// Mark form as having changes
unsavedChanges.setDirty();

// Clear after saving
unsavedChanges.setClean();

// Check current state
if (unsavedChanges.isDirty()) { ... }

Note: Modern browsers show a generic message like "Changes you made may not be saved" — custom messages are ignored for security reasons.
